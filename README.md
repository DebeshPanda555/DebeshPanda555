# 👨‍💻 About Me

I'm a final year B.Tech undergrad at VSSUT who spends most of my time thinking about two things: how money moves reliably through distributed systems, and how real-time state stays consistent when multiple users interact concurrently.

Right now I'm deep in both — **PayFlow** is a full-stack digital wallet and payment system where I've had to solve real problems: idempotency under retries, transaction consistency across concurrent writes, and WebSocket sync that doesn't fall apart at load. **DoodleVerse** came from wanting to understand what it takes to build something collaborative at low latency — turned out Go was the right call there.

I've also spent time at **Outlier AI** training LLM systems (improved output consistency 18%, built Python eval pipelines that increased throughput 22%) and at **Smarted Innovations** doing systems-level data analysis — 40+ test scenarios from 10K+ events, reduced bugs 28%.

I'm not chasing breadth. I want to go deep on backend systems, get good at distributed design, and ship things that hold up under pressure.

Full-stack payment system built with TypeScript and PostgreSQL. The core engineering challenge was ensuring strong consistency under concurrent operations — idempotent APIs, transaction logging for retries, real-time updates over WebSockets, and failure-scenario testing that actually breaks things on purpose.
